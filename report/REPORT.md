## 1 Инструмент управления проектной деятельностью

### 1.1 Формирование требований к выбору программного SCRUM-инструмента
Для ведения проекта сформированы следующие требования:

**Функциональные:**
1. Поддержка Product Backlog и Sprint Backlog.
2. Наличие доски задач с кастомными статусами.
3. Оценка задач в story points.
4. Разделение задач по спринтам и участникам.
5. История изменений и комментарии.

**Нефункциональные:**
6. Простой интерфейс для небольшой команды.
7. Работа в браузере без сложного администрирования.
8. Бесплатный тариф/пробный период.

### 1.2 Обзор существующих SCRUM-инструментов
Для выбора инструмента рассмотрены:
- Jira
- Kaiten
- YouGile
- Trello

Критерии сравнения:
- поддержка Scrum-артефактов;
- визуализация процесса;
- оценка задач;
- удобство для команды;
- доступность бесплатного тарифа.

В рамках учебного проекта выбран инструмент: **Trello** (или эквивалент, фактически использованный командой), т.к. он быстро настраивается и покрывает базовые требования Scrum-процесса.

## 2 Реализация проекта

### 2.1 Описание разрабатываемой системы
Разработана веб-система **«Ремонт 4-48»** для информационной поддержки ремонта техники.

#### Целевой бизнес-процесс
1. Заказчик приносит технику в сервис.
2. Менеджер оформляет заявку и назначает исполнителя.
3. Исполнитель выполняет диагностику и формирует перечень работ/деталей.
4. Менеджер согласует стоимость с заказчиком.
5. При согласии: склад резервирует детали, исполнитель выполняет ремонт.
6. При отказе: взимается только стоимость диагностики.
7. После оплаты заявка закрывается.

#### Роли в системе
- Менеджер
- Исполнитель
- Управляющий складом
- Администратор

#### Технический стек
- Python 3.13
- Django 6
- SQLite
- Django Templates
- uv

#### Основные сущности БД
- Customer
- Device
- WorkOrder
- DiagnosticResult
- WorkItem
- Part (закупочная и продажная цена)
- StockItem
- PartReservation
- PartUsage
- Payment
- OrderStatusHistory

#### Статусы заявки
`new` → `assigned` → `diagnosed` → `awaiting_approval` → `approved/rejected` → `in_progress` → `completed` → `closed`
